# 21 — Reglas para Claude Code y Antigravity

## Reglas críticas
1. Leer `CLAUDE.md`, `DECISIONS.md` y `PROJECT_STATUS.md` antes de tocar código.
2. Trabajar por fases.
3. No avanzar sin pruebas.
4. Actualizar `PROJECT_STATUS.md` tras cambios importantes.
5. Registrar decisiones nuevas en `DECISIONS.md`.
6. No eliminar requisitos sin justificar.
7. Mobile-first antes que escritorio.
8. SEO antes que decoración.
9. Rendimiento móvil antes que animaciones pesadas.
10. Admin configurable antes que datos hardcodeados.
11. IA siempre revisable por humano.
12. Pagos, contratos, documentos y datos privados siempre con seguridad.

## Flujo recomendado
- Planificar.
- Implementar pequeño.
- Probar.
- Documentar.
- Continuar.

## No hacer
- No crear landing genérica.
- No usar textos de relleno definitivos.
- No crear contenido duplicado masivo.
- No exponer claves.
- No hacer deploy production sin validación.
