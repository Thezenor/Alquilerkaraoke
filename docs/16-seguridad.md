# 16 — Seguridad

## Requisitos
- Roles y permisos.
- 2FA para admin si es viable.
- Logs de cambios.
- Protección CSRF.
- Validaciones con Zod.
- Rate limiting.
- Protección formularios anti-spam.
- Sanitización de inputs.
- Seguridad de documentos privados.
- Control de acceso a galerías privadas.
- Variables de entorno seguras.
- No exponer claves API.
- Backups.

## Datos sensibles
- Documentos clientes.
- Contratos.
- Facturas.
- Firmas.
- Pagos.
- Chats.

Deben protegerse con permisos y URLs no públicas.
